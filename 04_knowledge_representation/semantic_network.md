# セマンティックネットワーク（意味ネットワーク）

## 要点
- **ノード（概念）とエッジ（関係）**でグラフ構造により知識を表現する古典的手法
- **IS-A関係**（上位・下位概念）、**HAS-A関係**（部分・全体）などを直感的に表現
- 視覚的・構造的な知識表現で、**推論は継承や探索**で実現（確率的推論ではない）

## 定義
**セマンティックネットワーク（意味ネットワーク、Semantic Network）**は、概念を**ノード（節点）**、概念間の関係を**エッジ（辺）**で表現する**グラフ構造**による知識表現手法。第二次AIブーム期（1980年代）に発展した記号的AIの代表的アプローチの一つ。

## 重要キーワード
- **ノード（Node）**: 概念・オブジェクトを表す節点（例：「犬」「動物」「哺乳類」）
- **エッジ（Edge）**: ノード間の関係を表すリンク（例：IS-A、HAS-A、PART-OF）
- **IS-A関係（分類関係）**: 上位・下位概念の関係（例：「犬 IS-A 哺乳類」）
- **HAS-A関係（所有関係）**: 属性・部分の関係（例：「犬 HAS-A 尾」）
- **PART-OF関係**: 部分・全体の関係（例：「エンジン PART-OF 車」）
- **継承（Inheritance）**: 上位概念の属性を下位概念が引き継ぐ仕組み
- **探索（Search）**: グラフを辿って関連知識を検索する推論方法

## 基本構造

### 表現例
```
          動物
         /    \
      IS-A    IS-A
       /        \
    哺乳類      鳥類
     /           \
   IS-A         IS-A
    /             \
   犬            スズメ
   |              |
 HAS-A          HAS-A
   |              |
  尾            羽
```

**説明**:
- 「犬 IS-A 哺乳類」: 犬は哺乳類の一種
- 「哺乳類 IS-A 動物」: 哺乳類は動物の一種
- 「犬 HAS-A 尾」: 犬は尾を持つ
- **継承**: 「犬」は「動物」の属性（例：「呼吸する」）を自動的に継承

### 推論の仕組み
1. **直接的推論**: エッジを辿って関係を確認（例：「犬は動物か？」→ IS-Aを辿って「はい」）
2. **継承による推論**: 上位概念の属性を下位に適用（例：「動物は呼吸する」→「犬も呼吸する」）
3. **探索による推論**: グラフを幅優先・深さ優先で探索

## セマンティックネットワークの特徴

### 利点
- ✅ **視覚的・直感的**: 知識構造が図で分かりやすい
- ✅ **柔軟な拡張**: 新しいノード・エッジを容易に追加可能
- ✅ **継承が自然**: 上位概念の属性を効率的に再利用
- ✅ **構造的知識表現**: 概念間の関係を明示

### 欠点
- ❌ **曖昧性**: 関係の意味が厳密でない場合がある（IS-Aの多義性など）
- ❌ **複雑性**: 大規模知識では管理が困難
- ❌ **推論の限界**: 複雑な論理推論には不向き（確率的推論もできない）
- ❌ **例外処理**: 例外的ケース（「ペンギンは鳥だが飛べない」）の扱いが難しい

## 試験での問われ方

**典型的な問題**:

**問題**: セマンティックネットワーク（意味ネットワーク）の説明として、**最も不適切な選択肢**を1つ選べ。

1. ✅ ノードとエッジを用いてグラフ構造で概念と関係を表現する  
   → **適切**。セマンティックネットワークの基本構造。

2. ✅ IS-A関係やHAS-A関係を用いて概念間の階層や属性を表現する  
   → **適切**。IS-A（上位・下位）、HAS-A（所有）は代表的な関係。

3. ✅ 上位概念の属性を下位概念が継承する仕組みを持つ  
   → **適切**。継承はセマンティックネットワークの重要な特徴。

4. ❌ **確率分布を用いて不確実性を含む推論を行う**  
   → **不適切（正解）**。これは**ベイジアンネットワーク**の説明。セマンティックネットワークは**決定的（確率なし）**。

**ひっかけポイント**:

| 手法 | 知識表現 | 推論方法 | 不確実性 | 時代 |
|------|---------|---------|---------|------|
| **セマンティックネットワーク** | グラフ（ノード+エッジ） | 継承・探索 | **なし（決定的）** | 第二次ブーム |
| **ベイジアンネットワーク** | 有向グラフ（確率付き） | **確率的推論** | **あり（確率分布）** | 1980年代以降 |
| **ルールベースシステム** | IF-THENルール | 前向き/後ろ向き推論 | なし（一部あり） | 第二次ブーム |
| **オントロジー** | 概念+関係+制約（形式化） | 論理的推論 | なし | 1990年代以降 |

### 混同しやすい概念との違い

**1. セマンティックネットワーク vs ベイジアンネットワーク**
- **セマンティック**: 決定的、グラフ構造、継承・探索
- **ベイジアン**: **確率的**、条件付き確率、不確実性の推論

**2. セマンティックネットワーク vs ルールベースシステム**
- **セマンティック**: グラフ構造、視覚的、関係性重視
- **ルールベース**: **IF-THENルール**、テキスト形式、論理的手続き重視

**3. セマンティックネットワーク vs オントロジー**
- **セマンティック**: 非形式的、柔軟、視覚的
- **オントロジー**: **形式的定義**、論理的整合性、標準化（OWL、RDF）

**4. セマンティックネットワーク vs ニューラルネットワーク**
- **セマンティック**: 記号的AI、明示的知識、人間が設計
- **ニューラル**: **統計的学習**、暗黙的知識、データから学習

### 典型的な不適切選択肢

以下は**セマンティックネットワークには該当しない**説明（他手法の特徴）:

| 不適切な説明 | 実際はどの手法か | 理由 |
|-------------|----------------|------|
| 確率分布を用いて不確実性を推論 | **ベイジアンネットワーク** | セマンティックは決定的 |
| IF-THENルールで知識を記述 | **ルールベースシステム** | セマンティックはグラフ構造 |
| 統計的機械学習でデータから学習 | **機械学習** | セマンティックは人手で構築 |
| 深層学習で特徴を自動抽出 | **ニューラルネットワーク** | セマンティックは記号的AI |
| 形式論理で厳密に定義 | **オントロジー** | セマンティックは非形式的 |

## 代表的な応用例

### 歴史的実例
- **WordNet**: 英単語の意味関係を構造化した大規模セマンティックネットワーク
- **ConceptNet**: 常識知識を表現するオープンソースネットワーク
- **医療診断システム**: 症状と疾患の関係を表現

### 現代での位置づけ
- 知識グラフ（Knowledge Graph）の基礎概念として継承
- Google Knowledge Graph、DBpedia等に影響
- オントロジー工学の前身

## 補足

### 実務的観点
- **知識ベース設計**: 構造化された知識表現が必要な場合の基礎
- **自然言語処理**: 単語の意味関係の理解に応用
- **推薦システム**: アイテム間の関係性をグラフで表現
- **医療AI**: 疾患・症状・治療法の関係を構造化

### フレーム問題との関係
セマンティックネットワークも記号的AIの一種であり、**フレーム問題**（何が変化し何が不変かを列挙する計算量爆発）の影響を受ける。例外処理（「ペンギンは鳥だが飛べない」）の表現に課題がある。

### 第二次AIブームとの関連
1980年代の第二次AIブームでは、エキスパートシステムとともにセマンティックネットワークが注目されたが、知識獲得のボトルネックと推論の限界により衰退した。現在は知識グラフやオントロジーに進化している。
