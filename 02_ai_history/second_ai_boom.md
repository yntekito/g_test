# 第二次AIブーム（1980年代）

## 要点
- **エキスパートシステム**の実用化により到来（1980年代）。専門家の知識をIF-THENルールで表現し推論を実行
- 代表例は医療診断システム**MYCIN**、化学分析システム**DENDRAL**。特定領域で専門家レベルの性能を実現
- **知識獲得のボトルネック**（知識の記述コスト大）と**常識の欠如**により衰退。汎用性の限界が露呈

## 定義
第二次AIブームは、1980年代に**エキスパートシステム（Expert System）**の実用化により起きたAIブーム。専門家（エキスパート）の知識を「IF（条件）-THEN（結論）」形式のルール（プロダクションルール）として記述し、推論エンジンで自動推論を行うシステムが特徴。医療診断、化学分析、コンピュータ設定等の特定領域で成功を収めたが、知識獲得の困難さと汎用性の欠如により1990年代に再び冬の時代へ。

---

## 第二次AIブームの背景

### 第一次ブームの失敗から
- **第一次ブーム（1950～60年代）**: 探索・推論で単純な問題は解決できたが、現実の複雑な問題には無力
- **課題**: 「知識」をどう扱うかが不明確だった
- **転換点**: 「推論能力」より「知識の量と質」が重要と認識

### 知識表現と推論への着目
- 人間の専門家は膨大な知識と経験に基づいて判断している
- その知識を形式的に表現し、コンピュータで推論できれば専門家レベルの判断が可能

---

## エキスパートシステム（Expert System）★最重要

### 定義
専門家（エキスパート）の知識を体系的に取り込み、その知識に基づいて推論を行う人工知能システム。**第二次AIブームの中核技術**。

### 基本構成

エキスパートシステムは以下の3つの主要コンポーネントで構成：

```
┌─────────────────────────────────┐
│   【知識ベース（Knowledge Base）】  │
│   IF-THENルールの集合              │
│   IF 条件 THEN 結論               │
└─────────────────────────────────┘
           ↓↑
┌─────────────────────────────────┐
│   【推論エンジン（Inference Engine）】│
│   ルールを適用して推論を実行        │
│   前向き推論・後ろ向き推論          │
└─────────────────────────────────┘
           ↓↑
┌─────────────────────────────────┐
│   【ユーザーインターフェース】       │
│   質問応答、説明機能               │
└─────────────────────────────────┘
```

#### 1. 知識ベース（Knowledge Base）
専門家の知識を**IF-THENルール（プロダクションルール）**として記述。

**ルールの形式**:
```
IF 条件1 AND 条件2 THEN 結論
IF 患者の体温 > 38度 AND 咳がある THEN インフルエンザの可能性あり
```

**特徴**:
- 宣言的な知識表現（手続きではなく事実とルール）
- 人間が読める形式
- 追加・修正が比較的容易

#### 2. 推論エンジン（Inference Engine）
知識ベースのルールを適用して推論を実行。

**主な推論方式**:

**前向き推論（Forward Chaining / データ駆動）**:
- 既知の事実から出発し、ルールを適用して結論を導出
- 流れ: データ → ルール適用 → 新たな結論 → 繰り返し
```
例: 症状（発熱、咳）→ ルール適用 → 「風邪の可能性」→ さらにルール適用 → 診断
```

**後ろ向き推論（Backward Chaining / ゴール駆動）**:
- 仮説（ゴール）から出発し、それを支持する証拠を探索
- 流れ: 仮説 → 必要な条件を逆算 → 証拠を確認
```
例: 仮説「インフルエンザ」→ 必要条件「発熱、関節痛」→ 患者に質問して確認
```

#### 3. ユーザーインターフェース
- 質問応答形式での対話
- **説明機能**: なぜその結論に至ったかをルールの連鎖で説明（透明性）

## 代表的なエキスパートシステム

### MYCIN（マイシン）★最重要・試験頻出

**概要**:
- **開発**: 1970年代、スタンフォード大学
- **領域**: 感染症の診断と抗生物質の処方推薦
- **知識**: 約600のIF-THENルール
- **性能**: 専門医と同等レベルの診断精度（約65%の正答率、専門医も同程度）

**動作例**:
```
システム: 「患者の体温は？」
医師: 「39.5度」
システム: 「白血球数は？」
医師: 「15000」
システム: 「髄膜炎の可能性が高い。推奨抗生物質はペニシリン」
システム: 【説明】「ルール23: IF 高熱 AND 高白血球 THEN 細菌感染、ルール57: IF 細菌感染 AND 髄膜刺激症状 THEN 髄膜炎...」
```

**特徴**:
- **確信度（Certainty Factor）**: ルールに確率的な重みをつけて不確実性を扱う
- **説明機能**: 推論過程を医師に提示（ブラックボックスではない）
- **実用化には至らず**: 法的責任問題、医師の受容性の問題

**意義**:
- エキスパートシステムの可能性を示した象徴的システム
- 知識ベース型AIの代表例として教科書に必ず登場

---

### DENDRAL（デンドラル）

**概要**:
- **開発**: 1960年代後半、スタンフォード大学
- **領域**: 質量分析データから化学構造を推定
- **意義**: 世界初の実用的エキスパートシステム

**動作**:
- 質量分析の結果（分子量、断片パターン）を入力
- 可能な化学構造を生成し、データと照合
- 最も妥当な分子構造を提案

**成果**:
- 有機化学者レベルの性能
- 実際の研究で使用され、論文にも貢献

---

### XCON（R1）

**概要**:
- **開発**: 1980年、カーネギーメロン大学 → DEC社で実用化
- **領域**: VAXコンピュータシステムの構成設計
- **規模**: 約10,000のルール
- **効果**: 年間数千万ドルのコスト削減

**成功の理由**:
- 明確に定義された限定領域
- 企業での実用に成功（商業的成功の初期例）

---

## 第二次AIブームの技術的基盤

### 知識表現の手法
- **プロダクションルール（IF-THEN）**: 最も一般的
- **フレーム**: 構造化されたデータ表現（オブジェクト指向の先駆け）
- **セマンティックネットワーク**: 概念間の関係をグラフで表現
- **述語論理**: 数理論理学に基づく厳密な表現

詳細は [04_knowledge_representation/](../04_knowledge_representation/) を参照

### 第五世代コンピュータプロジェクト（日本）

**概要**:
- **期間**: 1982～1992年
- **目標**: 並列推論マシンによる知識処理の高速化
- **投資**: 約540億円
- **成果**: 部分的な技術成果はあったが、目標は未達成

**目標技術**:
- 論理型言語Prologベースの並列推論
- 知識ベースシステムの高速化
- 自然言語理解

**結果**:
- 技術的には部分的成功だが、実用化には至らず
- 第二次AIブーム終焉の象徴的プロジェクト

## 第二次AIブームの衰退原因

### 1. 知識獲得のボトルネック★試験頻出

**問題**:
- 専門家の知識を形式的なルールに変換する作業が極めて困難
- **知識工学者**（Knowledge Engineer）が専門家にインタビューして手作業でルール化
- 1つのシステムに数年かかる場合も

**具体的課題**:
- 専門家が知識を言語化できない（暗黙知）
- ルール数が膨大（数千～数万）になると矛盾が発生
- ルールのメンテナンスコストが高い

**比喩**:
料理の達人に「どうやって美味しく作るか」を全てレシピに書き起こさせるようなもの。感覚的なコツや経験は言語化困難。

---

### 2. 常識の欠如★試験頻出

**問題**:
- エキスパートシステムは専門知識のみを持ち、**常識（Common Sense）**を欠く
- 限定された領域外の質問には全く対応できない

**具体例**:
```
医療診断システム（MYCIN）:
Q: 「患者は車に轢かれて頭部外傷があります」
A: 「体温は？白血球数は？」（感染症の診断しかできない）

常識があれば: 「まず外科医を呼ぶべき」と判断できるが、システムには無理
```

**フレーム問題との関連**:
- 関連する知識の範囲を特定できない
- 常識的推論ができない
- 詳細は [03_ai_definition/frame_problem.md](../03_ai_definition/frame_problem.md) を参照

---

### 3. 汎用性の欠如

**問題**:
- 各エキスパートシステムは特定領域専用
- 領域が少し変わるだけで全く使えない
- 知識の再利用が困難

**例**:
- 感染症診断システムは内科疾患には使えない
- 1つのドメインに特化しすぎ

---

### 4. 学習能力の欠如

**問題**:
- ルールは人間が手作業で記述
- システム自身がデータから学習する機能はない
- 新しい知識の追加は常に手作業

**第三次ブームとの対比**:
- 第二次: 知識は手動で記述（エキスパートシステム）
- 第三次: データから自動学習（機械学習・深層学習）

---

## 重要キーワード

- **第二次AIブーム**: 1980年代、エキスパートシステムの時代
- **エキスパートシステム（Expert System）**: 専門家の知識を取り込んだAIシステム★最重要
- **知識ベース（Knowledge Base）**: IF-THENルールの集合
- **推論エンジン（Inference Engine）**: ルールを適用して推論
- **IF-THENルール（プロダクションルール）**: 「IF 条件 THEN 結論」形式の知識表現
- **前向き推論（Forward Chaining）**: データから結論へ
- **後ろ向き推論（Backward Chaining）**: 仮説から証拠へ
- **MYCIN（マイシン）**: 医療診断システム、最も有名なエキスパートシステム
- **DENDRAL（デンドラル）**: 化学分析システム、世界初の実用的ES
- **知識獲得のボトルネック**: 知識の形式化が困難でコスト大★衰退の主因
- **常識の欠如**: 限定領域外の推論ができない★衰退の主因
- **第五世代コンピュータ**: 日本の国家プロジェクト（1982～1992）

## 試験での問われ方

### 典型的な出題パターン

1. **第二次AIブームの特徴**: 「1980年代のAIブームの中核技術は？」→ **エキスパートシステム**
2. **専門家の知識を取り込んだシステム**: 「専門家の知識を取り込んだ人工知能システムは？」→ **エキスパートシステム**★頻出
3. **代表例**: 「医療診断のエキスパートシステムは？」→ **MYCIN**
4. **知識表現**: 「エキスパートシステムの知識表現形式は？」→ **IF-THENルール（プロダクションルール）**
5. **衰退原因**: 「第二次AIブームが終わった理由は？」→ **知識獲得のボトルネック、常識の欠如**
6. **推論方式**: 「データから結論を導く推論方式は？」→ **前向き推論（Forward Chaining）**

### ひっかけポイントと違いの整理

#### 第一次 vs 第二次 vs 第三次ブームの比較

| 項目 | 第一次（1950～60年代） | 第二次（1980年代） | 第三次（2010年代～） |
|------|---------------------|------------------|------------------|
| **中核技術** | 探索・推論 | **エキスパートシステム** | 深層学習 |
| **知識の扱い** | ほぼなし | 手動で記述 | データから自動学習 |
| **代表例** | チェス、定理証明 | **MYCIN, DENDRAL** | AlphaGo, GPT |
| **衰退原因** | トイプロブレム | **知識獲得のボトルネック** | （継続中） |

**試験で問われるポイント**:
- 「専門家の知識」「IF-THENルール」→ 第二次ブーム
- 「データから学習」「深層学習」→ 第三次ブーム

#### エキスパートシステムの混同注意

| 項目 | エキスパートシステム | 機械学習（第三次） |
|------|---------------------|-------------------|
| 知識の獲得 | 人間が手動で記述 | データから自動学習 |
| 知識の形式 | IF-THENルール | パラメータ（重み） |
| 説明性 | 高い（ルール提示可） | 低い（ブラックボックス） |
| 汎用性 | 低い（特定領域） | 高い（転移学習可） |
| 時代 | 1980年代 | 2010年代～ |

#### 代表システムの識別

| システム名 | 領域 | 覚え方 |
|-----------|------|--------|
| **MYCIN** | 医療診断 | MY（私の）＋医学（Medicine） |
| **DENDRAL** | 化学分析 | DENDRO（樹）＋構造解析 |
| **XCON** | コンピュータ設計 | X＋CONfiguration |

**問題パターン**:
- 「MYCINの領域は？」→ **医療診断（感染症）**
- 「DENDRALの領域は？」→ **化学分析（質量分析）**
- 「世界初の実用的エキスパートシステムは？」→ **DENDRAL**
- 「最も有名な医療診断システムは？」→ **MYCIN**

#### 選択肢で出やすいパターン

**正解となる記述**:
- ✅ 「専門家の知識をIF-THENルールで表現」
- ✅ 「推論エンジンで自動推論を実行」
- ✅ 「MYCINは医療診断のエキスパートシステム」
- ✅ 「知識獲得のボトルネックが衰退の原因」

**誤答の典型**:
- ❌ 「データから自動的に知識を学習」→ 機械学習の特徴
- ❌ 「ニューラルネットワークで実現」→ 第三次ブームの技術
- ❌ 「探索と推論が中心技術」→ 第一次ブームの特徴
- ❌ 「汎用的な常識推論が可能」→ 逆、常識の欠如が問題

## 補足

### 実務的観点

**現代への影響**:
- **ルールベースシステムは現在も活用**: 
  - 金融の信用審査（明確な基準が必要）
  - 医療の診断支援（説明責任が重要）
  - 法律の判例検索
- **機械学習との組み合わせ**: 
  - ルールで事前フィルタ、機械学習で精密判定
  - ハイブリッドシステム
- **説明可能AI（XAI）への再評価**:
  - 深層学習のブラックボックス問題
  - エキスパートシステムの説明性が再評価

### 教訓

**第二次ブームからの学び**:
1. **知識の形式化は困難**: 暗黙知を明示知にするのは本質的に難しい
2. **常識の重要性**: 専門知識だけでは不十分、常識的推論が必要
3. **学習の必要性**: 手動記述では限界、自動学習が必須（第三次ブームへ）

### 歴史的意義

- **AI研究の方向転換**: 「推論」から「知識」へ
- **実用化の萌芽**: 初めて商業的価値を示した
- **限界の認識**: 記号処理だけでは知能を実現できないことが明確に
