# フレーム問題・常識の問題

## 要点
- **フレーム問題**：行動の前後で「何が変化し、何が変化しないか」を列挙する計算量爆発の問題。AIの根本的課題。
- **常識の問題**：人間が持つ膨大な一般常識（物理法則・社会規範等）をAIに獲得させることの困難さ。
- **シンボルグラウンディング問題**：記号と現実世界の意味の対応付けの困難さ。記号だけでは意味が理解できない。

## 定義

### フレーム問題（Frame Problem）
AIが行動を実行する際、**その行動によって影響を受ける事柄と影響を受けない事柄を全て列挙する必要があり、これが計算量的に不可能**になる問題。AI研究における最も根本的な哲学的問題の一つ。

**具体例**：ロボットが部屋でバッテリーを取るタスク
- 変化すること：ロボットの位置、バッテリーの位置、手の状態
- 変化しないこと：部屋の色、天井の高さ、時刻、外の天気、他の惑星の位置…（無限に列挙）

人間は無意識に「関連する変化」だけに注目できるが、AIには困難。

### 常識の問題（Common Sense Problem）
**人間が日常的に持っている膨大な一般常識**（物理法則、因果関係、社会規範、文脈理解等）を、AIに獲得・表現させることの困難さ。

**例**：
- 「コップを逆さにすると水がこぼれる」（物理常識）
- 「図書館では静かにする」（社会常識）
- 「雨の日は傘を持つ」（実用常識）

これらを全て明示的にプログラムすることは不可能。

### シンボルグラウンディング問題（Symbol Grounding Problem）
**記号（シンボル）と現実世界の対象・意味を結びつける**ことの困難さ。記号操作だけでは、その記号が何を意味するのか理解できない。

**例**：「犬」という記号
- 辞書定義：「イヌ科の哺乳類」
- しかし「イヌ科」「哺乳類」も記号
- 実世界の犬（視覚・触覚・音等）との対応付けが必要

## 重要キーワード
- **フレーム問題（Frame Problem）**：行動による変化・不変を全て列挙する計算量爆発。
- **フレーム公理（Frame Axiom）**：変化しない事柄を記述する論理式（膨大になる）。
- **資格問題（Qualification Problem）**：行動の前提条件を全て列挙できない問題。
- **ラモニカル問題（Ramification Problem）**：行動の間接的影響を全て予測できない問題。
- **常識の問題（Common Sense Problem）**：一般常識をAIに習得させる困難さ。
- **常識推論（Common Sense Reasoning）**：日常的な知識に基づく推論。
- **シンボルグラウンディング問題（Symbol Grounding Problem）**：記号と現実世界の意味の対応付け。
- **中国語の部屋（Chinese Room）**：記号操作≠理解の思考実験（サール）。
- **Cyc プロジェクト**：人間の常識を論理式で記述する試み（1984年～）。
- **知識獲得のボトルネック**：専門知識・常識を手作業で入力する限界。

## 詳細

### フレーム問題の詳細

#### 古典的な例：イェール射撃問題
```
状況:
- 銃に弾が入っている
- 待機する（行動1）
- 射撃する（行動2）

問題:
- 待機後も銃に弾が入っているか？ → 明示的に記述しないと不明
- 射撃後、銃の色は変わったか？ → 無関係だが考慮する必要
- 地球は自転し続けているか？ → 無限の不変事項
```

#### フレーム問題の本質
1. **表現上の問題**：変化しない事柄を全て記述すると膨大
2. **推論上の問題**：全ての可能性を検証すると計算不可能
3. **哲学的問題**：「関連性」をどう判断するか

#### 人間との違い
- **人間**：文脈から自動的に関連する情報のみ抽出（無意識）
- **AI**：関連性の判断基準を明示的に与える必要

### 常識の問題の詳細

#### 常識の種類

| 種類 | 説明 | 例 |
|------|------|-----|
| **物理常識** | 物理法則の直観的理解 | 重力、固体・液体の性質、因果関係 |
| **社会常識** | 社会規範・慣習 | 挨拶、礼儀、法律、道徳 |
| **実用常識** | 日常生活の知恵 | 雨なら傘、暑いなら薄着 |
| **心理常識** | 他者の心理の理解 | 怒っている人には近づかない |
| **言語常識** | 言葉の背景知識 | 比喩、皮肉、文脈依存の意味 |

#### 常識獲得の困難さ
1. **膨大な量**：人間が持つ常識は数百万～数億の事実
2. **暗黙知**：明示的に言語化されていない
3. **文脈依存**：状況によって適用が変わる
4. **例外だらけ**：一般ルールに多数の例外

#### Cyc プロジェクト
- **1984年開始**、ダグラス・レナト主導
- 人間の常識を論理式で手作業入力
- 数百万の事実と推論規則を記述
- **結果**：限定的成功、汎用的常識獲得には至らず
- **教訓**：知識を明示的に記述する限界

### シンボルグラウンディング問題

#### 記号循環の問題
```
辞書で「犬」を調べる:
犬 → イヌ科の哺乳類
イヌ科 → 食肉目の一科
哺乳類 → 脊椎動物の一綱
脊椎動物 → 背骨を持つ動物
...（記号の循環）

実世界との対応がない
```

#### グラウンディングの方法
1. **感覚運動的グラウンディング**：視覚・聴覚等で実物と対応
2. **身体性（Embodiment）**：身体を持つロボットで環境と相互作用
3. **マルチモーダル学習**：画像・音声・テキストを統合学習（CLIP等）

#### 現代の解決アプローチ
- **深層学習**：画像認識で「犬」の視覚パターンを学習
- **マルチモーダルモデル**：テキスト＋画像で記号と実世界を結合
- **大規模言語モデル**：膨大なテキストから暗黙知を統計的に獲得

### 3つの問題の関係

```
【フレーム問題】
行動の影響範囲を特定できない
    ↓ 関連
【常識の問題】
「何が関連するか」の判断に常識が必要
    ↓ 関連
【シンボルグラウンディング問題】
常識を記号だけで表現しても意味が理解できない
```

全て「人間が無意識にできることをAIに実装する困難さ」という共通点。

## 実例

### 例1：フレーム問題
```
タスク: キッチンで料理を作る

考慮すべき変化:
- 食材の位置・状態
- 火の有無
- 調理器具の温度
- 時間経過
...

考慮不要な変化（だが明示的に記述が必要？）:
- 隣の部屋の本の位置
- 窓の外の天気
- 地球の自転
- 月の満ち欠け
→ 無限に列挙
```

### 例2：常識の問題
```
「太郎は傘を持って出かけた」

人間の推論:
- 雨が降っている（または予報）
- 濡れたくない
- 傘で雨を防げる
→ 自動的に理解

AIへの実装:
- 雨→濡れる（因果）
- 濡れる→不快（知識）
- 傘→雨を防ぐ（機能）
- 天気予報→傘持参（推論）
→ 全て明示的に記述が必要
```

### 例3：シンボルグラウンディング
```
記号操作: 「犬」「猫」は異なる記号として処理可能
しかし:
- 「犬」が何を指すのか理解していない
- 実物の犬を見せても「犬」と結びつかない

解決策:
- 画像認識: 犬の画像データで学習
- マルチモーダル: "dog"という単語と犬画像を同時学習
→ 記号と実世界を対応付け
```

## 試験での問われ方

### 典型問題
**問**: 「人間が持っている一般常識を人工知能に習得させることは困難である問題」について、最も適切なものを選べ。

- ✅ **正解**: 常識の問題（Common Sense Problem）
- ✅ **正解**: フレーム問題（広義の文脈では関連）
- △ **部分正解**: シンボルグラウンディング問題（記号と意味の対応付け）
- ❌ 誤答: チューリングテスト（知能の判定基準、常識獲得の問題ではない）
- ❌ 誤答: 中国語の部屋（理解の哲学的問題、常識獲得の困難さとは別）

### 問題の表現と対応

| 問題文のキーワード | 対応する問題 |
|-------------------|--------------|
| 「一般常識の習得が困難」 | **常識の問題** |
| 「変化する/しないことの列挙」 | **フレーム問題** |
| 「行動の影響範囲が不明」 | **フレーム問題** |
| 「記号と意味の対応付け」 | **シンボルグラウンディング問題** |
| 「辞書的な記号循環」 | **シンボルグラウンディング問題** |
| 「前提条件の列挙が困難」 | **資格問題** |

### 比較されやすい概念

| 概念 | 説明 | 違いのポイント |
|------|------|----------------|
| **フレーム問題** | 変化・不変の列挙困難 | 計算量爆発、関連性判断 |
| **常識の問題** | 一般常識の獲得困難 | 知識の膨大さ、暗黙知 |
| **シンボルグラウンディング** | 記号と意味の対応困難 | 記号操作≠理解 |
| **中国語の部屋** | 記号操作≠理解の思考実験 | 哲学的論証、実装問題でない |
| **チューリングテスト** | 知能の判定基準 | テスト手法、問題の種類でない |

### 引っ掛けポイント
1. **「常識」とあれば常識の問題**（最も直接的）
2. **「変化する/しない」とあればフレーム問題**
3. **「記号と意味」とあればシンボルグラウンディング問題**
4. **フレーム問題は広義**：常識が必要な背景問題として含むことも
5. **中国語の部屋との混同**：中国語の部屋は「理解」の哲学的問題、常識獲得の困難さとは別
6. **Cycプロジェクトは常識の問題への挑戦**（結果は限定的）

### 関連する歴史的背景
- **1969年**：マッカーシー&ヘイズがフレーム問題を提起
- **1980年**：サールが中国語の部屋を提唱
- **1984年**：Cycプロジェクト開始（常識の論理化）
- **1990年**：シンボルグラウンディング問題が提起
- **2010年代～**：深層学習で暗黙知の統計的獲得へシフト

## 補足

### 現代AIでの対応

#### 深層学習による「常識」の獲得
- **大規模言語モデル（LLM）**：膨大なテキストから統計的に常識を学習
- **BERT/GPT**：文脈理解、ある程度の常識推論が可能
- **Vision-Language Model（CLIP等）**：画像とテキストでグラウンディング

#### 限界
- 統計的「常識」は真の理解ではない可能性
- 学習データにない状況では失敗
- ハルシネーション（事実でない生成）

### 実務的観点
1. **ドメイン限定**：特定領域に常識を限定すれば実用可能（医療AI、将棋AI等）
2. **人間との協調**：AIが苦手な常識判断は人間が補完
3. **継続学習**：新しい常識を追加学習で獲得
4. **説明可能性**：判断根拠を提示し、人間が常識チェック

### フレーム問題の実用的回避策
- **状態変化の明示的記述**：限定されたドメインで変化を定義
- **デフォルト推論**：「通常は変化しない」と仮定
- **非単調論理**：新情報で結論を修正可能に
- **深層学習**：統計的に関連性を学習（根本解決ではない）

### 哲学的意義
- **AIの限界を示す問題**：記号的AIの根本的課題
- **人間の知能の特殊性**：無意識の常識判断の重要性
- **強いAIへの障壁**：真の知能には常識理解が不可欠
